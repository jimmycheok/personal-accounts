# Release v2.1 — Duplicate Records & Mileage Rate Fix

**Date:** 2026-03-27
**Branch:** release/v2.1
**Base commit:** fb0b7d9

## Summary

Adds a "Duplicate" action for invoices, expenses, and mileage entries — opens the create form pre-filled with the original record's values so users can quickly create similar transactions without re-entering everything. Also fixes a mismatch between the frontend and backend mileage deduction rates, aligning both to the LHDN-approved tiered schedule (RM 0.60/km for the first 200 km/month, RM 0.40/km thereafter).

## Spec Changes

- **Mileage deduction rate**: The documented rate of RM 0.25/km was incorrect. The backend has always used RM 0.60/km (configurable via `MILEAGE_RATE_PER_KM` env var). The correct LHDN rate is tiered: RM 0.60/km for the first 200 km/month, RM 0.40/km thereafter. Note: the tiered calculation (switching to RM 0.40 after 200 km) is not yet implemented in the backend — it applies a flat rate per entry. This is a known limitation for a future release.
- **Invoice duplication**: The `InvoiceForm` page now accepts `location.state.duplicate` (React Router state) to pre-fill the form. This is a frontend-only change; no new API endpoints were added.
- **Expense prefill contract**: `AddExpenseModal`'s `prefill` prop now accepts `category_id` (number) and `notes` (string) in addition to the existing OCR fields (`vendor`, `description`, `amount`, `date`).
