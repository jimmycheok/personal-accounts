# Release v2.2 — Document Preview & Upload Constraints

**Date:** 2026-03-30
**Branch:** release/v2.2
**Base commit:** a899d0f

## Summary
Adds in-app document preview for images and PDFs across the Documents page and all attachment panels. Restricts all document uploads system-wide to PDF and image files only (JPG, PNG, GIF, WEBP), enforced on both the API and frontend. Also fixes a floating-point rounding issue in the mileage deductible preview.

## Spec Changes
- **New API route**: `GET /documents/:id/preview` — serves file with `Content-Disposition: inline` for PDF and image MIME types; returns 415 for unsupported types
- **Upload restriction**: `POST /documents` now validates MIME type server-side — only `application/pdf`, `image/jpeg`, `image/png`, `image/gif`, `image/webp` are accepted; all other types return 400
- **Frontend accept constraints**: All file inputs (Documents upload, AttachmentsPanel, AddExpenseModal) now only accept `.pdf,.jpg,.jpeg,.png,.gif,.webp` — CSV, XLSX, XLS, DOC, DOCX no longer accepted for document storage
