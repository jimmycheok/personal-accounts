# Release v1.3 — Code Quality, Shared Tax Constants & Minor Fixes

**Date:** 2026-03-18
**Branch:** release/v1.3
**Base commit:** `f434f5a`

## Summary

v1.3 is a cleanup and quality release. The Taxation page now imports tax brackets and the `calculateTax` function from the shared `@personal-accountant/shared` package instead of duplicating them inline. Synchronous filesystem calls in the OCR receipt handler are replaced with async equivalents. Dynamic imports of `EinvoiceSubmission` in the invoices controller are replaced with a static top-level import. Several minor UI and data-lookup bugs are fixed across Bank Reconciliation, Documents, Quotation Form, and Settings pages.

## Changed Files

| File | What changed |

## Bug Fixes

| Area | Fix |
|---|---|
| Bank Reconciliation | Ignore toggle failed silently — ID comparison now uses `String()` coercion on both sides |
| Bank Reconciliation | Ignore button passed wrong ID (cell ID vs row ID) — now passes `row.id` |
| Documents | Row click opened wrong document when filters were active — lookup changed from positional index to `find()` by ID |
| Invoices controller | `EinvoiceSubmission` was dynamically imported on every call — moved to static import for consistency and performance |
| OCR receipt | Synchronous filesystem calls blocked the event loop — replaced with async equivalents |

## Spec Changes

| Area | Change |
|---|---|
| Taxation page | Tax brackets and calculation logic are no longer duplicated in the frontend; the shared package (`@personal-accountant/shared`) is the single source of truth. `TAX_BRACKETS_AY2024` rates are stored as decimals (e.g. `0.21` not `21`), so display code multiplies by 100. |
| `calculateTax` return value | Now returns `{ tax, effectiveRate }` object (not just a number). `effectiveRate` is a decimal (e.g. `0.15` for 15%). |
