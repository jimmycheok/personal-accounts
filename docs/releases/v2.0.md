# Release v2.0 — Chart of Accounts, General Ledger & Financial Reports

**Date:** 2026-03-27
**Branch:** release/v2.0
**Base commit:** `f58aefc`

## Summary

v2.0 adds a full double-entry accounting layer to the application. A Chart of Accounts (35 pre-seeded Malaysian sole proprietor accounts mapped to Borang B D1-D20), General Ledger with journal entries, and financial reports (Profit & Loss, Balance Sheet, Trial Balance) are now available. Every financial transaction (invoice send, payment, expense, credit note, mileage trip) passes through a GL Review Modal where users choose debit/credit accounts before the transaction is saved. An optional AI assistant (Claude Sonnet via Anthropic SDK) can suggest the correct accounts on demand. Year-end closing automation transfers net profit to Retained Earnings.

## Spec Changes

### New database tables (3)
- `accounts` — Chart of Accounts: `code` (unique), `name`, `account_type` (enum: asset/liability/equity/revenue/expense), `sub_type`, `borang_b_section`, `is_system`, `is_active`, `parent_id` (self-ref)
- `journal_entries` — GL entries: `entry_date`, `reference_number` (unique, format JE-YYYYMM-0001), `description`, `is_auto`, `source_type`+`source_id` (polymorphic), `status` (enum: draft/posted), `posted_at`
- `journal_entry_lines` — Entry lines: `journal_entry_id` (FK, cascade delete), `account_id` (FK, restrict delete), `debit` DECIMAL(15,2), `credit` DECIMAL(15,2)

### New API routes
- `GET/POST /api/v1/accounts` — list/create accounts
- `PUT /api/v1/accounts/:id` — update account (system accounts: code/type locked)
- `GET /api/v1/accounts/:id/ledger` — account ledger with running balance
- `GET/POST /api/v1/journal-entries` — list/create journal entries
- `POST /api/v1/journal-entries/ai-suggest` — AI-powered GL suggestion (accepts `{ type, data }`)
- `GET /api/v1/journal-entries/:id` — journal entry detail with lines
- `POST /api/v1/journal-entries/:id/post` — post a draft entry
- `DELETE /api/v1/journal-entries/:id` — delete draft manual entries only
- `GET /api/v1/reports/profit-loss?from=&to=` — P&L report (JSON)
- `GET /api/v1/reports/profit-loss/pdf?from=&to=` — P&L report (PDF)
- `GET /api/v1/reports/balance-sheet?as_at=` — Balance Sheet (JSON)
- `GET /api/v1/reports/balance-sheet/pdf?as_at=` — Balance Sheet (PDF)
- `GET /api/v1/reports/trial-balance?as_at=` — Trial Balance
- `POST /api/v1/reports/year-end-close?year=` — Year-end closing entries

### Changed API contracts
- `POST /invoices/:id/send` — now accepts optional `journal_lines` array in request body
- `POST /invoices/:id/mark-paid` — now accepts optional `journal_lines`
- `POST /invoices/:id/void` — now accepts optional `journal_lines`
- `POST /invoices/:invoiceId/payments` — now accepts optional `journal_lines`
- `POST /expenses` — now accepts optional `journal_lines`
- `POST /credit-notes/:id/send` — now accepts optional `journal_lines`
- `POST /credit-notes/:id/void` — now accepts optional `journal_lines`
- `POST /mileage` — now accepts optional `journal_lines`

### New environment variable
- `CLAUDE_API_KEY` — Anthropic API key for AI-powered GL suggestions (optional; AI suggest button won't work without it)

### GL Review workflow
All financial mutations (invoice send, payment, expense, credit note, mileage) now show a GL Review Modal before saving. The modal pre-fills smart default accounts based on transaction type. Users can modify accounts manually or click "AI Suggest" to get Claude-powered suggestions. "Skip GL Entry" allows proceeding without creating a GL entry.

## Upgrade Notes

1. **Run new migrations:**
   ```bash
   cd apps/api
   npx sequelize-cli db:migrate
   ```

2. **Seed Chart of Accounts:**
   ```bash
   npx sequelize-cli db:seed --seed 20260327000001-chart-of-accounts.cjs
   ```

3. **Generate retroactive journal entries** (optional — creates GL entries for all existing transactions):
   ```bash
   npx sequelize-cli db:seed --seed 20260327000002-retroactive-journal-entries.cjs
   ```

4. **New env var** (optional): Add `CLAUDE_API_KEY` to `.env` for AI-powered GL suggestions. The feature degrades gracefully — users can always select accounts manually.

5. **Install Anthropic SDK** (if not already):
   ```bash
   yarn add @anthropic-ai/sdk
   ```
